## PRD Summary（PRD 摘要）

- PRD 中的一句话产品价值：同一道题匿名呈现两个模型回答，voter 盲投后由 admin 结算并展示揭盲与 ELO 排行。
- PRD 中的目标用户：预置 voter `2510631109-user-demo` 与预置 admin `2510631109-admin-demo`。
- PRD 中最重要的非目标：不做注册、真实账号、真实模型 endpoint、前端美化、防刷、回放或智能体选手。

## v1 Summary（v1 摘要）

- v1 的核心设计：PRD v1 先定义 US-1、US-2；SPEC v1 定义匿名 A/B、一次投票与 ELO 规则。
- v1 中最清楚的一条验收标准：初始 ELO 均为 1500、A 胜、K=32 时，A 为 1516、B 为 1484，变化量代数和为 0。
- v1 中最容易跑偏的一处范围：v1 将投票与结算/揭盲合并，和参考规格的 `created -> answering -> ready -> voted -> scored` 状态机及 settle 边界不一致。

## 缺失的验收标准

- v1 缺少 `POST /battles` 的创建与 `created -> answering -> ready` 过程。
- v1 缺少 `POST /battles/{battle_id}/settle` 的独立结算、重复 settle 的 `409` 与明确的 A/B 随机位置要求。
- v1 的排行榜字段未完整覆盖 `name`、`wins`、`losses`、`draws`、`battles`。

## 缺失的错误路径

- v1 的正式错误表只有 6 类，未分别覆盖空/超长 prompt、相同 Contestant、超时、空回答、endpoint 不可达和重复 settle。
- 已在 SPEC v1.1 的 ERR-1 至 ERR-9 中补为 9 条可观察路径；所有失败路径均声明不改 ELO 或榜单。
