# ADR-001：课程 MVP 使用本地 JSON 原子写入

- 状态：已接受
- 日期：2026-07-20

## 背景

提交期需要一个可本地运行、可测试、可复查的 MVP。当前功能只需保存少量方案草案，没有并发用户和云端部署需求。

## 决策

使用 `data/launchspec.json` 存储项目；写入时先生成临时文件，再通过 `rename` 原子替换目标文件。运行时数据被 Git 忽略，保留 `data/.gitkeep` 维持目录。

## 备选方案

1. SQLite：查询能力与迁移更强，但引入额外驱动、schema/migration 维护和环境差异。
2. PostgreSQL：适合多人生产系统，但需要服务部署、连接凭据与运维，不符合当前最小范围。
3. 内存数组：实现最小，但刷新即丢失，无法满足编辑保存和 UAT。

## 后果

- 优点：无服务依赖、离线可运行、可直接复查数据格式。
- 代价：不支持多进程并发写入和复杂查询；这在 README、PRD 和 DESIGN 中被明确列为边界。
